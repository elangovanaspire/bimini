Cloning the machine[edit]

  • Clone the Bimini latitude machine on VMware vSphere/ESXi
  • Set an IP address /etc/network/interfaces

Optional

  • Set a MAC address - vmWare will do this automatically
  • Set DNS servers - Should be set, but double check to make sure we're using local ones.
  • add/remove username(s)

Re-Configure Linux, Apache, MySQL, WordPress[edit]

Grab a copy of Latitude-tools[edit]

  • Latitude-tools on Percipia's Gitlab
  • It's probably best to log into GitLab on your personal machine, grab a tar.gz archive, upload it to the server via SFTP, and untar it.
 cd latitude-tools.git

Modify config.cfg[edit]

  • You really only need to edit $ip, $hostname, and $mysql_pass variables.
  • Find $mysql_pass in /var/www/html/wp-config.php.
 # set machine ip address/hostname
 ip=""
 # set the hostname as site.percipia.net not simply site
 hostname=""
 interface="eth0"
 
 # set mysql db info
 mysql_path="/usr/bin/mysql"
 mysql_dump="/usr/bin/mysqldump"
 mysql_db="latitude"
 mysql_host="localhost"
 mysql_user="root"
 # find the sql db password in /var/www/html/wp-config.php
 mysql_pass=""
 
 # backup config
 wordpress_html="/var/www/html/"
 apache_config="/etc/apache2/"
 backup_dir="$HOME/backup-$(hostname)_$(date +%Y-%m-%d)"

Run the ./update_config.sh script[edit]

This script will automatically update the machine's Apache configuration, /etc/hosts file, hostname, and some SQL settings for WordPress. 

 ./update_config.sh

Update Permalinks[edit]

  • Sometimes you will have to update Permalinks (See: Settings Permalinks SubPanel)

Update Weather location[edit]

  • You can now do this using the WP interface when logged in as the admin user. Look for Weather API on the left menu.
  • If you'd rather update the database directly: 
 select * from wp_postmeta where meta_id = 6093;
 UPDATE wp_postmeta SET meta_value='bimini, bahamas' where meta_id = 6093;

Next steps[edit]

You can now begin modifying WordPress settings.

Create and restore database backups[edit]

Create up to the second backups[edit]

Use the latitude-tools' ./do_backups.sh script.

To do so (assuming you've extracted latitude-tools to ~/latitude-tools.git): 

 cd ~/latitude-tools.git
 ./do_backup.sh

In ~/backup_yyyy-mm-dd/ there should be a backup of:

  • the WordPress database - latitude.sql.bz2
  • WordPress files /var/www/html/ - wp_html.tar.gz
  • and Apache configuration /etc/apache2/ - apache_conf.tar.gz

Note: If you're doing more than one backup a day you might want to change $backup_dir in config.cfg to backup_dir="/root/backup_$(date +%s)"

Restoring database backups[edit]

In the latest version of Latitude Tools, you can restore database backups with the db-restore.sh script. 

 ./db-restore.sh /path/to/latitude.sql # (or latitude.sql.bz2)

The script will prompt you for the database you would like to import into (the default database is listed in config.cfg) and call do_backup.sh if you choose to restore the db into an existing db.

If you'd like you can also do it through the MySQL console: 

 cd ~/backup_yyyy-mm-dd/
 bzip2 -d latitude.sql.bz2
 /usr/bin/mysql --host="localhost" --user="root" -p

In the MySQL console, drop, create, and import the latitude db: 

 drop database latitude;
 create database latitude;
 use latitude;
 source ./latitude.sql;

WordPress Notes[edit]

Modify WordPress[edit]

  1. Visit http://$hostname/wp-admin/ and log in with the following credentials
    1. username: admin
    2. password: <ask someone>
  2. Remove old users

Now, to update all other options, yay!

Wordpress Admin Menu[edit]

When one logs into Wordpress, there will be a list of menu items on the left hand side. The names of these menu items are stored in multiple places.

custom-post-type menu[edit]

To change custom-post-type "Add New" button names, Menu names you need to update CPT UI.

To change the displayed names of "slugs," or custom post types, in CPT UI:

  • CPT UI >> Add Edit Post Types >> Edit Post Types >> Plural Label, Menu Names, Add New

other menus[edit]

Most of the other Menu modifications can be done using the "Menu Editor" plugin with a "Super Admin" account (as defined by Advanced Access Manager -- or AAM).

Sorting Items[edit]

The iOS app sorts differently based on the type of post that is published.

Events/ Promotions[edit]

These types of posts are sorted by date published. To change the sort order either:

  1. Delete and re-add the tiles in the correct order
  2. Update the post_date and post_date_gmt for each post id

Since, we're running a crippled WordPress install, to update post_date and post_date_gmt one must update the database directly. 

 SELECT `post_date` from `wp_posts` where `ID` = 'xxxxxxx';
 UPDATE `wp_posts` SET `post_date` = '1002-06-24 14:00:18' WHERE `ID` = 'xxxxxxx';
 
 SELECT `post_date_gmt` from `wp_posts` where `ID` = 'xxxxxxx';
 UPDATE `wp_posts` SET `post_date_gmt` = '1002-06-24 14:00:18' WHERE `ID` = 'xxxxxxx';

Activity Guide[edit]

These types of posts are sorted by Title. To sort these, simply prefix the title with a digit (or two, if there are more than 9 posts).

Configure email (WP SMTP)[edit]

  • use smtp-relay.gmail.com as the mail server. See: Google Apps SMTP settings for more details.
  • If we are sending mail from another IP address (i.e. a machine outside the Chicago office,) we will need to configure Allowed senders and Authentication methods in Google Apps.
    • Step 6 "In the Allowed senders section... Only addresses in my domains"
    • Step 7 "In the Authentication section... Only accept mail from the specified IP addresses—The system only accepts mail sent from these IP addresses as coming from your domains."
from: latitude@percipia.com
name: Percipia Latitude
host: smtp-relay.gmail.com
secure: none
port: 25 # if TLS: 587 , if SSL: 465
authentication: no

Note: This should only work on the internal network. If we install latitude on site in the future, we will have to change these settings.

Uploading images the lazy way[edit]

  1. Upload the images (via sftp) straight to /var/www/html/wp-content/uploads/ carefully replacing/overwriting the old ones.
  2. Regenerate the image thumbnails using wp-cli:
 cd /var/www/html/
 ~/wp-cli/wp-cli.phar --allow-root media regenerate --yes

Securing WordPress[edit]

On bimini.percipia.net we've tried to mitigate attacks.

  • Updated version of Ubuntu to LTS 14.04 for access to future security updates
  • Filesystem
    • Disabled caching of pages on any search engine via robots.txt (only for search engines that respect robots.txt)
    • Removed readme.html and license.html from root wordpress directory to prevent potential version leaking
    • TODO: Reset file/directory permissions to 644 and 755 respectively
    • Changed ownership to root:www-data
  • Installed security plugins in WordPress
    • Bad Behavior/ Bad Behaviour - if a client is acting like a bad robot, they'll be banned
    • Fail2ban
      • also installed fail2ban package from Ubuntu repos and configured it
      • WordPress login attempts are logged at /var/log/auth.log
      • 3 failed attempts (from outside of our LAN -- i.e. 10.x) will result in an auto ban
  • Apache
    • Disabled access to xmlrpc.php (can be used to launch ddos attacks) and wp-config.php (should not be accessed via http)
    • Only allowed access to wp-includes from inside wordpress

We might also want to consider:

  • Removing the 'admin' wordpress username and replacing it with latitudeadmin
  • Restricting access to wp-login.php and /wp-admin/
    • Adding HTTP authentication to those files, so we can track failed/successful attempts further
  • Restrict access to the server based on a country's IP blocks. If we don't have clients from a specific location, there is no reason to allow it to connect. Limiting the amount of potential connections reduces the vector for attack.
  • Restrict outbound packets and disable unnecessary commands.
  • Don't give the webserver process owner a home shell.

iOS App[edit]

Publishing[edit]

The basic steps to get our app into the app store.

Great, if a bit outdated, guide on how to submit an iOS app from start to finish:

  • http://www.raywenderlich.com/8003/how-to-submit-your-app-to-apple-from-no-account-to-app-store-part-1
  • http://www.raywenderlich.com/8045/how-to-submit-your-app-to-apple-from-no-account-to-app-store-part-2

Key Requirements

  1. Apple Developer Account - about threefifty...
  2. Provisioning Profile/ Distribution Profile - Required: app IDs that are recommended to be unique for each app in "reverse-domain name style" format (e.g. net.percipia.bimini)
  3. Running an App on your device - Required: UUID, set Development target to minimum iOS version the app can run on, set Bundle Identifier to be the exact same as app ID
  4. iTunes Connect - Agree to terms for Free Apps
  5. Submitting App - Required: App name, App description, App keywords (can only be changed at app updates), Copyright notice, SKU (minimum two digits, unique among all our applications), Primary Language, Reviewer notes (notes for the person reviewing the app), Rating Information (for age restrictions, etc) [1], App icon (512x512), [2] at least one App screenshot: "... sized 320×460 (no status bar); 320×480; 640×920 (retina, no status); 640×960 (retina); or, for landscape, 480×300 (no status bar); 480×320; 960×600 (retina, no status bar); or 960×640 (retina)."
  6. in iTunes Connect you need to add the new App under Manage Applications
    1. Add new App
    2. Set
      1. Primary Language
      2. Company Name
      3. SKU
      4. Bundle ID (Same as appID)
    3. Set Availability Date
      1. As soon as its approved,
      2. A date in the future, or
      3. After its approved (Select a date next year as it can be changed)
    4. Price Tier: Free
    5. Version Number: same as Xcode Project
    6. Upload Binary via Xcode: Requires you to agree to US cryptography export restrictions
  7. In Xcode
    1. choose iOS Device in scheme
    2. Project >> Archive
    3. Submit...
    4. iOS Developer Credentials
    5. Application Identity -> Select Provisioning Profile
    6. Finish
  8. Wait for approval - takes about a week

Notes

  1. http://stackoverflow.com/questions/23240816/select-a-frequency-intensity-level-for-each-apple-content-description-itunes-con
  2. https://developer.apple.com/library/ios/documentation/UserExperience/Conceptual/MobileHIG/IconMatrix.html#//apple_ref/doc/uid/TP40006556-CH27-SW1

Android App[edit]

Download[edit]

Firstly, you need to download the source from the FTP: kalahari:ftp.payoda.com:21 

get Crocs Source Code_Android_16-06-2015.zip

Note that this is ftp not sftp.

We're going to be using Eclipse for this example. Make sure to get the Android SDK.

Further note: We should be using the Gitlab for this.

Dependencies[edit]

Once Eclipse is started, go to Window > Android SDK Manager and get the latest Android API.

Note that there are packages called:

  • Android Wear ARM EABI v71 System Image
  • Android Wear Intel x86 atom System Image

These two(and their variations) can cause errors when starting Eclipse, so it is recommended not to install them.

Libraries[edit]

Now that we have the dependencies, it's time to import the project and its libraries. Make sure to get all of the projects. There are 7 in total.

We have to make sure that they can all see each other. Right click and go to properties for each project. Then select your latest Android API and make sure that all the proper libraries are references with there being a green check next to each of them. If the project is a library(all but the main one, CrocsCasino) then mark it as one. You may also have to make some changes in the Java Build Path.

Changes[edit]

Once your libraries are done, you may start making changes to the code. All of the following are in CrocsCasino.

  • AndroidMainfest.xml
    1. Is there anything to change here?
  • res/values/stings.xml
    1. Here you change the app_name
    2. Keep in mind that if this isn't changed, Android may think that this is the same app as in the Play Store.
  • res/drawable
    1. splash_screen.jpg needs to be changed. Make sure to keep the resolution and filename.
  • src/com.android.casino.utils/AlertMessageDialog.java
    1. Change the string parameter for SetTitle() to the proper title for alert dialogues.
  • In the various drawable-* folders, the logo specific images need to be changed.
  • Make sure to change logo_white.png
  • src/com.android.casino.utils/AppUtil.java
    1. Here change crocs.percipia.net to whatever WP install you are using
    2. You can comment out the Payoda stuff.

Note that while you are running the app, many of the images will come from the CMS. They can't be changed from the app.

Packaging[edit]

Right click > Export > Android > Export Android Application

It will ask you for the project name and the keystore. For testing you can make your own.

If Android Lint doesn't let you build, go into the "Problems" window and delete every error manually.

Installing[edit]

You may use an application called ES File Explorer to host an FTP in order to upload your .apk file. Afterwards, you simply navigate to it and install it.

It is recommended to use an emulator instead. This will make testing much faster and will allow stack traces to be viewed. An emulator comes with Eclipse although it can be difficult to get running. Try editing the device until you find one that works. Change the target to your desired API level. Provide sufficient RAM and storage.
